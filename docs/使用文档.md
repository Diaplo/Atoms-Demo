# 使用文档

## 1. 项目简介

本项目是一个 AI Builder 演示系统。用户登录后，可以在聊天区输入需求，由模型生成 React 代码；代码会同步进入编辑器，并通过 Sandpack 在右侧实时渲染。系统还会保存聊天记录和代码版本，便于回看和恢复。

## 2. 运行环境

建议使用以下环境：

- Node.js 18 或更高版本
- npm 9 或更高版本
- 一个可用的 Supabase 项目
- 一个可用的 Postgres 数据库
- 一个支持 OpenAI-compatible 协议的模型服务

## 3. 环境变量

复制示例环境变量：

```bash
cp .env.example .env.local
```

需要配置的变量如下：

```bash
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
DATABASE_URL=your_database_url
NEXT_PUBLIC_APP_URL=http://localhost:3000
NEXT_PUBLIC_SANDPACK_BUNDLER_URL=https://sandpack-bundler.codesandbox.io
```

说明：

- `NEXT_PUBLIC_SUPABASE_URL`、`NEXT_PUBLIC_SUPABASE_ANON_KEY`：前端和 SSR 鉴权依赖
- `SUPABASE_SERVICE_ROLE_KEY`：服务端需要时使用，不能泄露
- `DATABASE_URL`：Drizzle 和业务数据持久化依赖
- `NEXT_PUBLIC_APP_URL`：本地或线上站点地址
- `NEXT_PUBLIC_SANDPACK_BUNDLER_URL`：Sandpack bundler 地址，默认使用官方服务

## 4. 本地启动

1. 安装依赖

```bash
npm install
```

2. 初始化数据库

```bash
npm run db:push
```

3. 启动开发环境

```bash
npm run dev
```

4. 打开浏览器访问 `http://localhost:3000`

## 5. 首次使用流程

### 5.1 注册或登录

- 未登录时，根路径会自动跳转到登录页
- 注册使用 Supabase Auth
- 如果开启邮箱确认，注册后需要先完成邮箱验证

### 5.2 进入工作区

登录后会进入 Dashboard，页面由四个区域组成：

- 左侧：聊天区
- 中间：代码编辑器
- 中间右侧：Preview 预览
- 最右侧：版本历史

### 5.3 配置模型

如果没有配置模型，聊天区会提示先填写：

- API Key
- Base URL
- Model ID

注意：

- 这组配置目前只保存在浏览器本地
- 切换浏览器、清空本地存储或更换设备后需要重新填写

### 5.4 发起 NewChat 并生成代码

可通过两种方式发起生成：

- 直接输入你的需求
- 点击对话区示例气泡，将提示词填入输入框后发送

发送后，系统会：

1. 创建或绑定当前聊天
2. 调用 `/api/chat` 流式获取模型输出
3. 从回复中提取带 `filepath` 标记的代码块
4. 将代码应用到编辑器和 Preview
5. 保存聊天消息
6. 自动创建一个代码版本

## 6. 编辑器与 Preview

### 6.1 编辑器

- 默认运行时文件包括 `/App.tsx`、`/index.tsx`、`/styles.css`
- 若生成结果包含多个文件，编辑器会按当前运行时显示可编辑文件
- 你可以直接在编辑器中修改代码，Preview 会重新构建

### 6.2 Preview

- Preview 基于 Sandpack `react-ts` 模板
- 默认走官方 bundler
- 如果网络较慢、模型输出异常或 bundler 请求失败，可能出现预览超时

### 6.3 Refresh

- Refresh 用于重新拉起当前预览会话
- 若外部 bundler 服务不可用，仅刷新按钮无法绕过外部故障

## 7. 聊天与版本历史

### 7.1 聊天记录

- 左侧 Sidebar 会展示历史聊天
- 点击历史聊天后，会加载对应消息和代码版本
- 默认聊天标题会根据首条用户消息自动生成

### 7.2 版本历史

当前支持：

- 每次 assistant 完成输出后自动创建版本
- 手动保存版本
- 恢复某个历史版本
- 删除非最新版本
- 导出代码归档

## 8. 部署说明

### 8.1 Vercel

部署到 Vercel 时，至少要同步配置以下环境变量：

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- `DATABASE_URL`
- `NEXT_PUBLIC_APP_URL`
- `NEXT_PUBLIC_SANDPACK_BUNDLER_URL`

### 8.2 请求时长

`/api/chat` 当前在代码中设置了：

```ts
export const maxDuration = 300
```

说明：

- 这表示函数期望最长运行 300 秒
- 线上最终时长还受 Vercel 套餐和 Fluid Compute 配置限制

## 9. 常见问题

### 9.1 注册时报 `email rate limit exceeded`

这是 Supabase Auth 邮件发送限流，不是前端表单校验错误。常见原因是短时间内反复触发注册确认邮件。

### 9.2 Preview 报 `TIME_OUT`

常见原因：

- Sandpack bundler 网络不稳定
- 生成代码本身有运行时错误
- 外部网络或浏览器策略阻断了 bundler 请求

建议优先检查：

- 浏览器 Console 报错
- Network 中 `sandpack-bundler.codesandbox.io` 请求状态

### 9.3 没有 API Key 无法发送消息

这是当前设计行为。系统会要求先配置模型信息，避免直接请求失败。

## 10. 当前限制

- 目前只支持单一 OpenAI-compatible 配置
- API Key 仅本地保存，不支持服务端加密托管
- Sandpack 依赖外部 bundler，线上稳定性受网络影响
- 版本对比能力还未接入主工作流
